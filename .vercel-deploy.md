# Déploiement sur Vercel

## 1. Lier le projet

- Va sur [vercel.com](https://vercel.com), connecte-toi, puis **Add New** → **Project**.
- Importe le repo Git (GitHub/GitLab/Bitbucket). Vercel détecte Vite automatiquement.

## 2. Variables d’environnement (à faire avant le premier build)

**Important :** ajoute les variables **avant** de lancer le build, ou dans **Settings** puis **Redeploy**.

Dans **Project → Settings → Environment Variables** (ou lors de l’import du projet), ajoute :

| Variable | Description | Où la trouver |
|----------|-------------|----------------|
| `VITE_SUPABASE_URL` | URL du projet Supabase | Supabase → **Settings** → **API** → **Project URL** |
| `VITE_SUPABASE_ANON_KEY` | Clé anon publique | Supabase → **Settings** → **API** → **Project API keys** → `anon` `public` |

Coche **Production** (et **Preview** si tu utilises les previews). Sans ces variables, le build peut passer mais l’app ne pourra pas se connecter à Supabase en prod.

## 3. Variables optionnelles

| Variable | Description |
|----------|-------------|
| `VITE_DEEPSEEK_API_KEY` | Clé API DeepSeek (chatbot landing + prédictions IA) |
| `VITE_APP_URL` | URL de l’app en prod (ex. `https://ton-app.vercel.app`) pour les liens/emails |

## 4. Build

- **Build Command** : `npm run build` (défini dans `vercel.json`).
- **Output Directory** : `dist`.
- **Install Command** : laisser par défaut (`npm install`).

Après chaque déploiement, les routes (`/`, `/login`, `/admin`, etc.) sont servies par `index.html` (SPA) grâce aux `rewrites` dans `vercel.json`.
