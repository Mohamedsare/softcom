# Déploiement sur Vercel

## 1. Lier le projet

- Va sur [vercel.com](https://vercel.com), connecte-toi, puis **Add New** → **Project**.
- Importe le repo Git (GitHub/GitLab/Bitbucket). Vercel détecte Vite automatiquement.

## 2. Variables d’environnement (obligatoires)

Dans **Project → Settings → Environment Variables**, ajoute au minimum :

| Variable | Description | Exemple |
|----------|-------------|---------|
| `VITE_SUPABASE_URL` | URL du projet Supabase | `https://xxx.supabase.co` |
| `VITE_SUPABASE_ANON_KEY` | Clé anon Supabase | `eyJhbGc...` |

Pour les définir : **Supabase** → ton projet → **Settings** → **API** (Project URL et anon public key).

## 3. Variables optionnelles

| Variable | Description |
|----------|-------------|
| `VITE_DEEPSEEK_API_KEY` | Clé API DeepSeek (chatbot landing + prédictions IA) |
| `VITE_APP_URL` | URL de l’app en prod (ex. `https://ton-app.vercel.app`) pour les liens/emails |

## 4. Build

- **Build Command** : `npm run build` (défini dans `vercel.json`).
- **Output Directory** : `dist`.
- **Install Command** : laisser par défaut (`npm install`).

Après chaque déploiement, les routes (`/`, `/login`, `/admin`, etc.) sont servies par `index.html` (SPA) grâce aux `rewrites` dans `vercel.json`.
